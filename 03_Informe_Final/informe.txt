# INFORME FINAL DE AUDITORÍA

## GlobalTrade Manufacturing

---

# 1. INTRODUCCIÓN

GlobalTrade Manufacturing es una empresa multinacional del sector manufacturero que opera en múltiples países y cuenta con una infraestructura tecnológica compleja que integra sistemas de Tecnologías de la Información (IT) y Tecnologías Operativas (OT).

En este contexto, se realizó una auditoría enfocada en la **seguridad de bases de datos y la aplicación ERP**, con el objetivo de identificar vulnerabilidades, evaluar configuraciones y analizar riesgos asociados a la protección de la información crítica.

---

# 2. OBJETIVO DE LA AUDITORÍA

Evaluar la seguridad de la base de datos y de la aplicación ERP de GlobalTrade Manufacturing, específicamente en:

* Identificación de vulnerabilidades en la base de datos
* Evaluación de controles de acceso
* Revisión de respaldos
* Análisis de configuraciones del servidor
* Evaluación de la seguridad de la aplicación ERP

---

# 3. ALCANCE

La auditoría se enfocó en:

* Servidor ERP (simulado): 10.50.20.10
* Base de datos asociada al ERP
* Aplicación web del ERP

---

# 4. METODOLOGÍA

Se utilizó un enfoque práctico basado en pruebas controladas, utilizando las siguientes herramientas:

* SQLMap (análisis de inyección SQL)
* OWASP ZAP (escaneo de vulnerabilidades web)

Las actividades realizadas incluyeron:

1. Pruebas de inyección SQL
2. Validación manual de vulnerabilidades
3. Revisión de usuarios y privilegios
4. Evaluación de respaldos
5. Verificación de configuraciones
6. Escaneo de la aplicación ERP

---

# 5. RESULTADOS DE LA AUDITORÍA

---

## 5.1 Hallazgos en Base de Datos

---

### DB-01: Vulnerabilidad de SQL Injection

**Descripción:**
Se identificó una vulnerabilidad de tipo SQL Injection en la aplicación ERP, permitiendo manipular consultas a la base de datos.

**Evidencia:**

* Prueba manual mediante inyección: `' OR '1'='1`
* Logs de SQLMap

**Impacto:**
Alto – acceso no autorizado a información sensible

**Probabilidad:**
Alta

**Riesgo:**
Crítico

**Recomendación:**

* Implementar consultas preparadas (prepared statements)
* Validar y sanitizar entradas de usuario

---

### DB-02: Uso de credenciales por defecto

**Descripción:**
Se identificó el uso de credenciales débiles o por defecto en la base de datos.

**Impacto:**
Alto

**Recomendación:**

* Implementar políticas de contraseñas seguras
* Rotación periódica de credenciales

---

### DB-03: Privilegios excesivos

**Descripción:**
El usuario administrador (root) posee privilegios totales sin restricciones.

**Impacto:**
Alto

**Recomendación:**

* Aplicar principio de mínimo privilegio
* Crear roles específicos

---

### DB-04: Falta de respaldos automatizados

**Descripción:**
No se evidenció un sistema automatizado de respaldos ni pruebas de restauración.

**Impacto:**
Alto

**Recomendación:**

* Implementar backups automáticos
* Validar restauraciones periódicas

---

### DB-05: Configuración de acceso a la base de datos

**Descripción:**
Se identificaron configuraciones que podrían permitir acceso no controlado.

**Impacto:**
Medio

**Recomendación:**

* Restringir acceso por IP
* Configurar firewall

---

---

## 5.2 Hallazgos en Aplicación ERP

---

### APP-01: SQL Injection en formularios

**Descripción:**
La aplicación no valida correctamente las entradas del usuario.

**Impacto:**
Crítico

---

### APP-02: Falta de validación de entradas

**Descripción:**
Los formularios permiten entrada de datos maliciosos.

**Impacto:**
Alto

---

### APP-03: Configuración insegura

**Descripción:**
La aplicación se encuentra en un entorno con configuraciones vulnerables.

**Impacto:**
Medio

---

### APP-04: Falta de controles de seguridad web

**Descripción:**
Se detectaron posibles vulnerabilidades mediante escaneo con OWASP ZAP.

**Impacto:**
Alto

---

# 6. ANÁLISIS DE RIESGO

La combinación de vulnerabilidades identificadas representa un riesgo significativo para GlobalTrade Manufacturing, especialmente en:

* Exposición de propiedad intelectual
* Manipulación de datos de producción
* Accesos no autorizados
* Interrupción de operaciones

---

# 7. CONCLUSIONES

La auditoría evidenció debilidades críticas en la seguridad de la base de datos y la aplicación ERP de GlobalTrade Manufacturing.

Las vulnerabilidades identificadas, especialmente SQL Injection y la mala gestión de accesos, representan un riesgo elevado que debe ser mitigado de forma prioritaria.

---

# 8. RECOMENDACIONES GENERALES

* Implementar controles de validación de entradas
* Aplicar principio de mínimo privilegio
* Fortalecer políticas de autenticación
* Automatizar respaldos
* Realizar auditorías periódicas
* Implementar monitoreo de seguridad
